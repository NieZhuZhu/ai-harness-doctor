## Why this matters

Efficacy is the project's strongest differentiator, but the current score gate
trusts any stored task records without proving which `AGENTS.md`, task set, or
repository facts produced them. The self-bootstrap result was generated on
2026-07-08 when `AGENTS.md` was 3.7KB; the current file is 12KB and contains
many new safety/release/MCP contracts. Yet CI still reports 12/12 because
`eval --score` recomputes pass rate from the old answers only.

A premium doctor must distinguish a current efficacy result from a stale green
artifact. Add deterministic evidence fingerprints to produced/regraded results,
and an explicit verification mode that fails closed when the current task file
or evidence files differ.

## Steps

### Step 1: Define a deterministic evidence manifest

Add stdlib-only helpers to `scripts/eval_run.py`:

- SHA-256 of the exact task-file bytes;
- a sorted mapping of repository-relative evidence file → SHA-256;
- a schema version for the metadata object.

Expose a repeatable CLI flag such as `--evidence FILE` for run, matrix, regrade,
and score/verify paths. Resolve each evidence path against `--workdir` when one
is supplied; reject missing files and paths outside that root. Store only
normalized repository-relative names and digests.

Recommended result shape:

```json
"evidence": {
  "schemaVersion": 1,
  "tasks": {"path": "benchmark/self-eval/tasks.json", "sha256": "..."},
  "files": [{"path": "AGENTS.md", "sha256": "..."}]
}
```

If a tasks file lives outside workdir for a legitimate use, preserve a stable
logical name supplied by the CLI rather than serializing an absolute path.

**Verify**: pure tests prove output is stable across cwd, dictionary ordering,
and file mtimes.

### Step 2: Stamp every result-producing path

Have single-run, multi-round, matrix, and regrade output include the manifest
whenever `--evidence` is supplied. Regrade must recompute the task digest and
evidence digests; it must not blindly retain stale metadata from the input.

Preserve byte-compatible legacy output when no evidence flag is supplied unless
adding the metadata unconditionally is proven safe. Do not change health math.

**Verify**: end-to-end tests cover run, matrix, and regrade manifests; existing
legacy-shape tests remain green.

### Step 3: Add explicit freshness verification

Add a score/verify flag such as:

```bash
ai-harness-doctor eval --score results.json \
  --tasks tasks.json --workdir . --evidence AGENTS.md \
  --require-current-evidence --fail-under 80
```

When `--require-current-evidence` is set:

- missing evidence metadata is a failure;
- task digest mismatch is a failure;
- missing/new/different evidence entries are a failure;
- health is evaluated only after freshness succeeds.

Use a distinct, documented non-zero exit code that does not collide with
health-under-threshold (5) or regression (6), e.g. 7. Print the mismatched
logical paths/digest class, never file contents.

Without the explicit flag, old results remain scoreable for backward
compatibility.

**Verify**: tests cover current, changed AGENTS, changed tasks, missing metadata,
path escape, and legacy non-strict score behavior.

### Step 4: Refresh self-eval honestly

Expand `benchmark/self-eval/tasks.json` to cover the high-value contracts added
since 2026-07-08 (at minimum installer state safety, report-family completeness,
MCP read-only/error policy, release versioning, and isolated HOME). Produce
answers using the repository's documented manual protocol if no real runner is
available, clearly labeling that evidence boundary.

Offline-regrade the current answers and stamp:

- `benchmark/self-eval/tasks.json`;
- current root `AGENTS.md`.

Update `benchmark/self-eval/README.md` with date, task count, method, and the
exact freshness verification command. Do not update the public headline
benchmark unless a comparable real runner experiment was performed.

**Verify**: refreshed result scores ≥80 and passes strict evidence verification;
changing one byte in a temp copy of AGENTS.md fails freshness.

### Step 5: Make this repo's PR gate enforce current evidence

Update only the self-bootstrap `.github/workflows/harness-drift.yml` to call the
strict freshness command with current tasks and `AGENTS.md`. The committed
result exists in this repo, so remove the soft “if file exists” skip here.

Keep generic shipped templates optional. If documentation/templates show how a
consumer opts into eval gating, add `--require-current-evidence` only when the
example also identifies tasks/evidence; never make the default template fail
for repositories without a benchmark.

Add structural tests ensuring the self workflow names the result, tasks, and
AGENTS evidence, and no longer silently skips its own committed file.

**Verify**: focused workflow tests and self gate pass.

### Step 6: Document the trust model and run full gates

Document:

- health says how stored records graded;
- freshness says whether they describe current evidence;
- CI should require both;
- fingerprints prove byte identity, not the honesty/quality of manually
  produced answers.

Synchronize all three READMEs and `SKILL.md`, mark Plan 015 DONE, then run the
full gate.
